--- tags: faq question: Where is my data? ---

Logically, data belongs to a domain, which is a collection of clones that belong to it. A clone is identified by a domain name, like example.m-ld.org.

The data is physically stored in the clone, which is typically embedded in the app that uses it. Different clone implementations may:

  • Keep the data in memory only
  • Persist the data directly to disk
  • Use some platform-specific local storage
