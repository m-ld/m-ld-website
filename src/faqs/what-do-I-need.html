--- tags: faq-disabled question: To use m-ld, what do I need? date: 2020-01-02 ---

m-ld is usually deployed embedded in your app so that access latency is zero. So, first you need a m-ld engine for your app's platform. Alternatively, you could use a clone deployed on a server (for example, using a Docker microservice).

You also need a supported message broker. If you'd like to deploy your own, you could use Eclipse Mosquitto, but it might be better to use a cloud service. Note the requirements for a message broker.

Finally, if your app requires data to be persisted after the last user has closed it, you will need a clone deployed on a server, which stores the data onto disk. A Docker microservice is ideal for this.
